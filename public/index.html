Secret Voting Demo

Instructions:

This dApp utilizes the Enigma Protocol Testnet to implement secret voting. The steps are as follows:
  1. Select an account. The default account is the first account (account 0) but you have the option of changing to a different account except for account number 9.
  2. Buy Voting Tokens. The current exchange rate is 1 Ether for 10 Voting Tokens.
  3. Stake voting tokens in the contract. The dApp will automatically approve the Voting Token transfer for you as well.
  4. Create a poll. Each poll is associated with a poll ID. There will be a label in the dashboard showing the poll ID of the current poll.
  5. Vote in the poll. In the current implementation, a user can only vote once and cannot change vote/weights during the poll.
  6. End the poll. After the poll period has expired, only the creator of a poll can end that poll.
    a. Once the poll has ended, a selected SGX node on the Enigma Network will privately tally the votes and update the result of the poll.
  7. Check the poll result. All poll metrics are public including the final status of the poll and the number of "yes"and "no" votes casted. However, due to secret voting, there is no way to match voters with their vote choice.
  8. Withdraw your tokens. You can only withdraw tokens that are not being used in other active polls.
